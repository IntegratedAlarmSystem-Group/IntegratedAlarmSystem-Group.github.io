org.eso.ias.plugin.publisher

PublisherBase 

abstract class PublisherBase extends MonitorPointSender

Base class for publishing data to the core of the IAS: received monitor points are queued to be sent when <UL> <LI>the time interval (#throttlingTime) elapses or <LI>the max allowed size of the buffer has been reached </UL> <P> PublisherBase gets all the values from the monitored system and saves them in a map (the buffer) until the throttling time expires or the max allowed size has been reached and only then sends all of them at once. The map allows to easily save only the last update value for each managed monitor point preventing a misbehaving plugin to fire tons of messages per time interval. Monitor point values are sent one by one to the core of the IAS. <P> Reaching the max size of the buffer, triggers the immediate sending of the values of the monitor points. The periodic thread will continue to run. <P> Life cyle: <UL> <LI>#start() is the first method to call to allow a correct initialization. <LI>#shutdown() must be called when down with this object to clean the resources </UL>
