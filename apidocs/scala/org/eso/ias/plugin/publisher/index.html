    The data structure encapsulating monitor point values and alarm retrieved from a monitored system and to be sent to the IAS core.

    The data structure encapsulating monitor point values and alarm retrieved from a monitored system and to be sent to the IAS core. It is composed of a global data structure (this one) plus a list of monitor points. <P> BufferedMonitoredSystemData is a java POJO that is easily serializable to a string that is what the plugin sends to the IAS core. <P> To improve network performances, whenever possible the plugin collects and sends the monitor points with one single message instead of one by one.

  5.  trait MonitorPointSender extends AnyRef

    The MonitorPointSender defines the interface to send monitor point values and alarms to the IAS core.

    The MonitorPointSender defines the interface to send monitor point values and alarms to the IAS core. <P> When a value is received from the monitored system and, if it is the case, filtered for noise, it is finally ready to be sent to the IAS core by invoking #offer(ValueToSend) <P> Implementers of this class, provides statistics collected during the last observation period, #getStats(). <P> <EM>Life cycle:</EM><BR> <UL> <LI>#setUp() must be called to allocate the resources needed for sending massages to the core of the IAS <LI>#tearDown() must be called to release the resources: no more messages can be sent after calling this method </UL> <P> Offering an item to a MonitorPointSender does not ensure that it is really sent to the final destination but the producer of monitor pont values is supposed to continue providing monitor point values at the proper time intervals. <BR> The sender is, in fact, totally decoupled from the core of the IAS and its functioning is not affected by the availability of the core or the transport mechanism. This allows to shutdown the transport framework or the core of the IAS without affecting the functioning of the plugin that results, de facto, totally decoupled from the IAS.

  6.  abstract class PublisherBase extends MonitorPointSender

    Base class for publishing data to the core of the IAS: received monitor points are queued to be sent when <UL> <LI>the time interval (#throttlingTime) elapses or <LI>the max allowed size of the buffer has been reached </UL> <P> PublisherBase gets all the values from the monitored system and saves them in a map (the buffer) until the throttling time expires or the max allowed size has been reached and only then sends all of them at once.

    Base class for publishing data to the core of the IAS: received monitor points are queued to be sent when <UL> <LI>the time interval (#throttlingTime) elapses or <LI>the max allowed size of the buffer has been reached </UL> <P> PublisherBase gets all the values from the monitored system and saves them in a map (the buffer) until the throttling time expires or the max allowed size has been reached and only then sends all of them at once. The map allows to easily save only the last update value for each managed monitor point preventing a misbehaving plugin to fire tons of messages per time interval. Monitor point values are sent one by one to the core of the IAS. <P> Reaching the max size of the buffer, triggers the immediate sending of the values of the monitor points. The periodic thread will continue to run. <P> Life cyle: <UL> <LI>#start() is the first method to call to allow a correct initialization. <LI>#shutdown() must be called when down with this object to clean the resources </UL>
